# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

Personal portfolio/CV website for Simon Robin (simrobin.fr). Content is in French.

## Commands

- **Dev server:** `pnpm start` or `pnpm dev`
- **Build:** `pnpm build`
- **Preview:** `pnpm preview`
- **Lint & format:** `pnpm lint` (runs biome check --fix on src/)
- **Format only:** `pnpm format` (runs biome format --write on src/)
- **Tests (watch):** `pnpm test`
- **Tests (CI/single run):** `pnpm test:ci`
- **Panda codegen:** `pnpm prepare` (runs automatically after install)

## Tech Stack

- **Astro 5** with static output, TypeScript strict
- **Panda CSS** for zero-runtime atomic styling with design tokens and semantic tokens (dark mode)
- **astro-icon** + Iconify for icons (local SVGs in `src/icons/`, standard icons from `simple-icons` and `lucide`)
- **Fontsource** for self-hosted Montserrat variable font
- **pnpm** package manager (pnpm@10, Node 22)
- **Vitest** + Astro Container API for component tests
- **Biome** for linting and formatting

## Architecture

Single-page static site:

- `src/pages/index.astro` — Main page composing all sections
- `src/layouts/Layout.astro` — HTML shell with `<head>`, meta tags, font import, View Transitions, dark mode anti-FOUC script
- `src/components/` — Astro components: `Header`, `Experience`, `Education`, `Footer`, `ThemeToggle`
- `src/content.config.ts` — Content Collection schemas (Zod) for experiences and education
- `src/data/experiences/*.yaml` — Experience entries (4 files)
- `src/data/education/*.yaml` — Education entries (2 files)
- `src/icons/` — Local SVG icons (10 files), used by `astro-icon`
- `src/assets/images/` — Optimized images (processed by Astro's Image component)
- `src/styles/global.css` — CSS layer declarations for Panda CSS
- `panda.config.ts` — Design tokens, semantic tokens (dark mode), breakpoints, global CSS
- `styled-system/` — Generated by Panda CSS (gitignored), provides `css()` utility

## Styling with Panda CSS

- Import `css` from `../../styled-system/css` in components
- Design tokens: `gray`, `lightGray`, `yellow`, `white` (colors); `light`, `regular`, `bold` (font weights)
- Semantic tokens: `bg`, `bg.header`, `bg.section`, `bg.card`, `text` — adapt to dark mode
- Breakpoints: `sm` (321px), `md` (768px), `lg` (1025px), `xl` (1281px)
- Dark mode: class-based (`.dark` on `<html>`), toggled via ThemeToggle component

## Content Collections

Experiences and education are YAML files with Zod schemas. Queried with `getCollection()` and sorted by `order` field.

## Pre-commit Hook

`script/hooks/pre-commit` auto-runs Biome on staged JS/TS/TSX/Astro files before commit.

## Style Conventions

- **Biome** handles linting and formatting
- Single quotes, trailing commas, 100 char line width, 2-space indent
- Config in `biome.json`
